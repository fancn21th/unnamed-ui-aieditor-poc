## 2026-03-10 问题 001：流式输出时工具栏按钮状态跳变

### 问题现象

Markdown 流式输出过程中，工具栏按钮会随着输出内容样式变化而高亮，表现为不是用户手动选中也会触发按钮 active 状态。

### 根因分析

流式写入调用 `insertContentAt` 时，会影响编辑器选区更新，导致工具栏状态判断使用了“程序写入中的选区”而不是“用户真实选区”。

### 解决方案

流式写入统一增加 `updateSelection: false`，禁止流式插入修改当前选区。

### 关键代码

文件：`src/hooks/use-streaming-editor.ts`

```ts
const insertOptions = {
  contentType: "markdown" as const,
  updateSelection: false,
}

editor.commands.insertContentAt(insertPos, msg.completedContent, insertOptions)
editor.commands.insertContentAt({ from, to }, msg.completedContent, insertOptions)
editor.commands.insertContentAt({ from, to }, msg.actualContent, insertOptions)
```

### 验证结果

流式输出期间，工具栏不再跟随流式内容样式自动跳变；按钮状态回归为用户选区驱动。

### 问题现象

2. 在流式处理钩子中增加原生选区清理：当编辑器未聚焦且存在非折叠 selection 时，执行 `removeAllRanges()`。

### 关键代码

## 2026-03-10 问题 003：Maximum update depth exceeded 循环更新报错

### 问题现象

接入 Bubble Menu/Floating Menu 与节点锁状态后，页面出现：

`Maximum update depth exceeded`

### 根因分析

`markdown3-editor.tsx` 中对 `editor.transaction` 事件监听后直接执行 `setNodeLockState`，而菜单组件在每次 React 重渲染时会触发插件选项更新并派发事务，导致“事务 -> setState -> 重渲染 -> 插件更新 -> 新事务”形成循环。

### 解决方案

1. 移除 `transaction` 事件监听，仅保留 `selectionUpdate` 同步锁状态。  
2. 增加状态一致性判断，避免相同状态重复 `setState`。  
3. `BubbleMenu/FloatingMenu` 的 `shouldShow` 改为直接基于入参 `state` 计算可见性，降低对 React 状态闭包的耦合。

### 关键代码

文件：`src/components/tiptap-templates/markdown3/markdown3-editor.tsx`

```ts
const syncNodeLockState = useCallback(() => {
  const next = getNodeLockState()
  setNodeLockState((prev) => {
    if (prev.available === next.available && prev.locked === next.locked) {
      return prev
    }
    return next
  })
}, [getNodeLockState])

editor.on("selectionUpdate", syncNodeLockState)
```

```ts
shouldShow={({ state }) => {
  if (!getNodeLockStateFromState(state).available) return false
  return !state.selection.empty
}}
```

### 验证结果

报错消失，编辑器不再出现循环更新；Bubble/Floating Menu 与锁定状态显示和交互正常。

## 2026-03-10 问题 005：气泡菜单按钮激活态未随选区状态正确高亮

### 问题现象

选区中已包含加粗、斜体、删除线等样式时，气泡菜单对应按钮没有高亮，和顶部工具栏表现不一致。

### 根因分析

按钮高亮依赖组件重渲染，但原实现仅在节点锁状态变化时触发本地状态更新；同一节点内切换不同文本样式时，锁状态不变，导致气泡菜单不会重渲染，按钮激活态滞后。

### 解决方案

1. 将气泡菜单抽离为独立组件 `markdown3/components/markdown3-bubble-menu.tsx`。  
2. 在气泡菜单组件中使用 `useEditorState` 订阅编辑器状态，实时计算 `bold/italic/strike/locked` 激活态。  
3. 按钮统一使用 `data-active-state="on|off"`，复用现有按钮激活样式规则。  
4. 继续保持“仅文本非空选区显示 Bubble Menu”的触发条件。

### 关键代码

文件：`src/components/tiptap-templates/markdown3/components/markdown3-bubble-menu.tsx`

```ts
const { isBold, isItalic, isStrike, nodeLockState } = useEditorState({
  editor,
  selector: ({ editor: currentEditor }) => ({
    isBold: currentEditor?.isActive("bold") ?? false,
    isItalic: currentEditor?.isActive("italic") ?? false,
    isStrike: currentEditor?.isActive("strike") ?? false,
    nodeLockState: currentEditor
      ? getNodeLockStateFromState(currentEditor.state, lockableNodeTypes)
      : { available: false, locked: false },
  }),
})
```

```tsx
<Button data-active-state={isBold ? "on" : "off"}>加粗</Button>
<Button data-active-state={isItalic ? "on" : "off"}>斜体</Button>
<Button data-active-state={isStrike ? "on" : "off"}>删除线</Button>
<Button data-active-state={nodeLockState.locked ? "on" : "off"}>锁定</Button>
```

文件：`src/components/tiptap-templates/markdown3/markdown3-editor.tsx`

```tsx
{editor && (
  <Markdown3BubbleMenu
    editor={editor}
    lockableNodeTypes={LOCKABLE_NODE_TYPES}
  />
)}
```

### 验证结果

重新选择不同样式文本时，气泡菜单按钮高亮可即时同步；组件职责划分更清晰，后续扩展菜单项更直接。

## 2026-03-11 新增功能 005：markdown-snapshot 扩展 Tiptap Snapshot 版本能力

### 功能介绍

在不改动 `markdown3` 的前提下，新增 `markdown-snapshot` 模板，并基于 `markdown3` 功能复制扩展出 Snapshot 版本管理能力。新增版本面板支持手动保存版本、拉取版本列表、切换自动快照、回滚到指定版本，并在面板中展示当前版本信息与同步状态，状态卡片提供快速跳转到版本列表入口。

### 问题与解决

1. **问题：`@tiptap-pro/extension-snapshot` 位于私有仓库，默认 npm registry 无法直接安装。**  
   **解决：** 改为运行时动态加载扩展。未安装时给出清晰提示，不阻断编辑器主功能；安装并鉴权后即可自动启用 Snapshot 能力。

2. **问题：Snapshot 需要协作 Provider 与凭据，否则版本命令不可用。**  
   **解决：** 增加 `@tiptap-pro/provider` 的运行时加载与配置校验；补齐 `@hocuspocus/provider`、`lib0` 依赖；未配置 `VITE_TIPTAP_APP_ID/VITE_TIPTAP_TOKEN` 时提示不可用，避免误操作。

3. **问题：Node 16 环境运行 ESLint 9 报 `structuredClone is not defined`。**  
   **解决：** 使用 Node 18+ 运行 lint，或升级本地 Node 版本再执行校验。

4. **问题：编辑器尚未挂载时访问 `editor.view.hasFocus` 导致页面报错。**  
   **解决：** 在 `useCursorVisibility` 中将 `editor.view` 与 `hasFocus` 访问包裹在 try/catch 中，未挂载时直接跳过处理，避免阻塞渲染。

5. **问题：协作 WebSocket 初始化失败时缺少明确状态信息。**  
   **解决：** 监听 Provider `status / authenticationFailed / close` 事件，在状态卡片展示连接状态与错误原因，便于定位连接失败原因。
