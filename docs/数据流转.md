# 数据流转（Word JSON → Tiptap Markdown → 增量回传）

## 1. 目标与约束

- 初始内容来自后端 Word 解析，默认锁定，不允许直接编辑。
- 用户只能对非初始节点新增、修改、删除。
- 需要保留并扩展节点属性：顺序、评论、修订、锚点、原始标签语义等。
- 导出阶段以“增量操作”为主，可附带最终快照用于审计与回放。

---

## 2. 阶段 A：后端原始格式（Word 解析结果）

后端可以继续输出“章节标题 => HTML 片段”，但建议带上文档元信息：

```json
{
  "docId": "DOC-001",
  "version": 1,
  "source": "word",
  "sectionsMap": {
    "2.3.1.自觉症状": "<h3><a name=\"_Toc248740760\">自觉症状</a></h3><p>...</p><a href=\"#_Toc248740760\">跳转</a>",
    "2.3.2.EYETEST时间(BUT)": "<p>...</p>"
  }
}
```

说明：

- 当前 `test-res.json` 是可用输入，但属于“业务 JSON”，不是编辑器可直接消费的文档结构。
- value 中存在 `<a href>`、`<a name/id>`、表格、图片、Word 导出样式等复杂信息。

---

## 3. 阶段 B：前端标准化格式（传给编辑器前）

把 `sectionsMap` 转成结构化数组，补齐节点元数据与可追踪标识：

```json
{
  "docId": "DOC-001",
  "baseVersion": 1,
  "contentType": "html-fragments",
  "sections": [
    {
      "id": "sec-2-3-1",
      "number": "2.3.1",
      "title": "自觉症状",
      "html": "<h3><a name=\"_Toc248740760\">自觉症状</a></h3><p>...</p><a href=\"#_Toc248740760\">跳转</a>",
      "origin": "initial",
      "locked": true,
      "order": 1,
      "meta": {
        "anchorId": "_Toc248740760",
        "sourceTag": "h3"
      }
    }
  ]
}
```

标准化动作建议：

- 按编号排序（`2.3.1` < `2.3.2`）。
- 清理重复章节（父节点包含子节点全文时优先保留叶子或按策略折叠）。
- 提取并结构化特殊信息：
  - `a[href]` → 链接属性
  - `a[name/id]` → 锚点属性
  - 原始标签信息 → `meta.sourceTag` / `meta.originalTag`

---

## 4. 阶段 C：Tiptap 渲染格式（ProseMirror JSON）

编辑器实际推荐使用 `doc` JSON 渲染，且把业务属性写入 `attrs` / `marks.attrs`：

```json
{
  "type": "doc",
  "content": [
    {
      "type": "heading",
      "attrs": {
        "level": 3,
        "nodeId": "n-1001",
        "sectionId": "sec-2-3-1",
        "origin": "initial",
        "locked": true,
        "anchorId": "_Toc248740760",
        "order": 1
      },
      "content": [{ "type": "text", "text": "自觉症状" }]
    },
    {
      "type": "paragraph",
      "attrs": {
        "nodeId": "n-1002",
        "sectionId": "sec-2-3-1",
        "origin": "initial",
        "locked": true
      },
      "content": [
        { "type": "text", "text": "查看锚点跳转：" },
        {
          "type": "text",
          "text": "跳转",
          "marks": [
            {
              "type": "link",
              "attrs": {
                "href": "#_Toc248740760",
                "target": null,
                "rel": "noopener noreferrer nofollow"
              }
            }
          ]
        }
      ]
    }
  ]
}
```

说明：

- `link` 通过 mark 的 `attrs.href` 承载。
- 标题跳转信息（锚点）建议落在节点 `attrs.anchorId`。
- 初始锁定由 `attrs.locked=true` 标识，并在事务层拦截修改。

---

## 5. 阶段 D：评论与修订的数据承载

### 5.1 评论

建议评论单独维护为 sidecar（便于并发与权限），节点仅保存索引：

```json
{
  "comments": [
    {
      "commentId": "c-9001",
      "nodeId": "n-1002",
      "range": { "from": 120, "to": 138 },
      "author": { "id": "u-1", "name": "Alice" },
      "content": "这个术语建议统一",
      "createdAt": "2026-03-11T10:30:00Z",
      "status": "open"
    }
  ]
}
```

### 5.2 修订

建议以“变更记录 + 节点快照”承载，便于审批与回放：

```json
{
  "revisions": [
    {
      "revisionId": "r-7001",
      "nodeId": "n-2001",
      "action": "update",
      "author": { "id": "u-2", "name": "Bob" },
      "createdAt": "2026-03-11T10:40:00Z",
      "snapshotBefore": { "type": "paragraph", "content": [{ "type": "text", "text": "原文" }] },
      "snapshotAfter": { "type": "paragraph", "content": [{ "type": "text", "text": "修订后" }] },
      "status": "pending"
    }
  ]
}
```

说明：

- `snapshotBefore/snapshotAfter` 可直接使用 ProseMirror 节点 JSON 结构。
- 若接入 Tiptap Snapshot 思路，可把每次修订视为一条“可回滚快照单元”。

---

## 6. 阶段 E：编辑完成后回传后端（增量优先）

推荐“增量操作 + 可选最终快照”的双轨回传：

```json
{
  "docId": "DOC-001",
  "baseVersion": 1,
  "editorVersion": 9,
  "changes": [
    {
      "changeId": "chg-1",
      "op": "insert",
      "nodeId": "n-3001",
      "parentSectionId": "sec-2-3-1",
      "origin": "user",
      "payload": {
        "type": "paragraph",
        "content": [{ "type": "text", "text": "用户新增内容" }]
      }
    },
    {
      "changeId": "chg-2",
      "op": "update",
      "nodeId": "n-2001",
      "origin": "user",
      "revisionId": "r-7001"
    }
  ],
  "comments": [
    {
      "commentId": "c-9001",
      "nodeId": "n-1002",
      "action": "create"
    }
  ],
  "snapshot": {
    "type": "doc",
    "content": []
  }
}
```

---

## 7. 字段落点建议（避免后续返工）

- 节点内（attrs）适合放：`nodeId/sectionId/origin/locked/order/anchorId`。
- sidecar 适合放：`comments/revisions/audit logs`（高并发、权限独立、查询频繁）。
- 导出接口至少区分：
  - `initial`（初始不可改内容）
  - `user`（用户新增可编辑内容）
  - `revision`（修订轨迹）

---

## 8. 对“Markdown 插件是否支持 JSON 渲染”的结论

- Tiptap 能渲染 ProseMirror JSON（`doc` 结构）。
- `@tiptap/markdown` 主要负责 markdown 字符串的导入/导出，不等于“业务 JSON 直接渲染”。
- 因此推荐链路：
  - 业务 JSON（Word 解析）→ 标准化 → ProseMirror JSON 渲染
  - 导出时按需产出 markdown / html / 增量 JSON
