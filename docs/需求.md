# Tiptap Markdown 编辑器高级扩展需求（整理版）


基本的在线编辑功能

可以渲染复制内容的html,图片,表格,标题等,并且对其进行锁定

可以渲染Agent生成的markdown内容

可以进行动态编号

支持修订功能,A修改B确认

支持将修订的内容导出,而且这部分内容要求能在导出到word时保留

可以识别并且保留复制内容的标签,在导出时提供,这部分内容会在导出时替换为原始的内容

可以支持引用,包括引用其他章节

支持动态识别锚点信息

支持文献角标引用,并且动态调整文献内容

## 0. 项目目标
构建一个支持高度定制业务逻辑的 Tiptap 编辑器，核心能力包括：
- 内容渲染与安全锁定
- 动态编号与跨章节引用
- 修订模式（Track Changes）
- 文献角标与参考文献联动
- 导出 Word 并保留修订痕迹

---

## 1. 内容渲染与安全锁定

### 1.1 粘贴 HTML/图片/表格并支持锁定
**需求描述**
- 支持复杂 HTML 内容粘贴渲染。
- 对特定块进行“可选中但不可编辑”的锁定。

**建议技术方案**
- 自定义 Node 扩展 + NodeView + `contenteditable="false"`。
- 通过 `parseHTML` / `renderHTML` 处理结构映射。
- 节点设置 `selectable: true`、`draggable: true`。

**涉及 API**
- `Node.create`
- `parseHTML`
- `renderHTML`
- `addNodeView`

**核心实现**
- 在 `parseHTML` 提高优先级，优先命中自定义节点。
- NodeView DOM 设置 `contenteditable="false"`。
- 允许块级移动/删除，不允许内部文本修改。

### 1.2 Agent 生成 Markdown 的渲染
**需求描述**
- 支持实时或异步渲染 AI 生成的 Markdown 文本。

**建议技术方案**
- 使用 Markdown 解析能力（官方 Markdown 插件或 markdown-it 管道）。
- 流式输出采用增量插入，保留空白与换行语义。

**涉及 API**
- `editor.commands.setContent`
- `editor.commands.insertContent`
- `parseOptions: { preserveWhitespace: true }`

## 2. 动态逻辑与文档结构

### 2.1 动态编号系统
**需求描述**
- 标题或列表编号可随结构变化自动重算。

**建议技术方案**
- 自定义 Heading 扩展 + ProseMirror 插件（`appendTransaction` / plugin state）。

**涉及 API**
- `addAttributes`
- `appendTransaction`
- `doc.descendants`

**核心实现**
- 为节点增加编号相关属性（如 `level/index`）。
- 文档变更后重算受影响范围编号并回写属性。
- 渲染层显示编号（DOM 注入或样式伪元素）。

### 2.2 跨章节引用与动态锚点
**需求描述**
- 引用其他章节并支持跳转，锚点位置动态更新。

**建议技术方案**
- 给章节节点注入唯一 ID（UUID）。
- 引用节点持有目标 ID，点击后跳转到目标。
- 维护“可引用锚点列表”。

**涉及 API**
- `addGlobalAttributes`
- `onUpdate`

---

## 3. 修订模式（Track Changes）

### 3.1 修订流程（A 修改，B 确认）
**需求描述**
- 记录插入与删除，需审核确认后落地。

**建议技术方案**
- 基于 Mark 的非破坏性修订层。
- 删除行为不直接删除，转为 deletion mark；新增自动 insertion mark。

**涉及 API**
- `Mark.create`
- `addCommands`
- `filterTransaction`

**核心实现**
- deletion mark：红色中划线，含 `authorId/timestamp`
- insertion mark：绿色下划线，含 `authorId/timestamp`
- 审核通过：
  - 删除标记 => 真正删除
  - 插入标记 => 移除 mark 转为普通文本

### 3.2 导出 Word 并保留修订
**需求描述**
- 导出 `.docx` 后，Word 仍能展示修订记录。

**建议技术方案**
- 后端将编辑器结构映射为 OpenXML。
- deletion -> `<w:del>`，insertion -> `<w:ins>`。

---

## 4. 复杂引用与文献系统

### 4.1 粘贴标签保留与导出替换
**需求描述**
- 保留原始标签语义，导出阶段可恢复替换。

**建议技术方案**
- `parseHTML` 捕获原始标签并存入 attributes（如 `data-original-tag`）。
- 导出序列化时按属性替换目标标签。

### 4.2 文献角标与参考文献联动
**需求描述**
- 文中角标按出现顺序自动编号，底部文献列表实时同步。

**建议技术方案**
- Inline 引用节点 + 扩展 storage 管理引用索引。
- 文档更新后重排角标并刷新参考文献列表。

**涉及 API**
- `addStorage`
- `onUpdate`

---

## 5. 验收标准（建议新增）
- 粘贴复杂 HTML 不丢结构，锁定块不可编辑但可移动/删除。
- 标题插入/删除后，编号自动正确更新。
- 跨章节引用可跳转，重排后不失效。
- 修订模式可记录作者和时间，支持逐条确认。
- 导出 Word 后保留修订可见性。
- 文献角标与参考文献列表顺序严格一致。
